Executive Summary

A Tinder-like preference learning system that uses existing embeddings and engagement data to efficiently learn user taste with minimal effort.

Core insight: With embedding-aware pair selection, only ~25-30 swipes are needed to build a meaningful preference model from a 1000-scene library, compared to thousands with random selection.

System Goals

  • Efficient learning: Minimize the number of user interactions needed to build a reliable preference model
  • Warm start: Leverage existing engagement data (o_count, play_count, ratings) as a Bayesian prior
  • Embedding-driven: Use pre-computed OpenCLIP embeddings for intelligent pair selection
  • Progressive refinement: Start with broad taste discovery, narrow to fine-grained preferences
  • Integration: Feed learned preferences back into the existing recommendation engine

Recommendation: Bayesian Bradley-Terry (Layer 1) + Active Learning with Embeddings (Layer 2). This combination provides the strongest warm-start capability, principled uncertainty tracking, and the fewest comparisons needed through intelligent pair selection.

Why Bayesian Bradley-Terry?

Advantage

Feature-Augmented Model

Learns a preference direction in embedding space: score(s) = p · e_s. Can rank scenes that have never been compared, based purely on their embeddings.

Advantage

Bayesian Uncertainty

Maintains full posterior p ~ N(μ, Σ). Knows exactly how confident each ranking is. Convergence detection is built-in: when σ drops below threshold, stop asking.

Progressive Refinement

Three phases of preference learning, from broad taste discovery to fine-grained boundary refinement.

Three-Phase Learning Process

Comparison Budget

Phase Swipes Cumulative Pair Selection Strategy What It Learns
Phase 1 6-10 6-10 Diverse: cluster representatives, round-robin Which taste categories preferred
Phase 2 12-18 18-28 Similar: high cosine-sim pairs within top clusters Fine ranking within preferred taste
Phase 3 1-3/session 28+ Uncertainty: max info-gain at decision boundary Edge cases, boundary precision

After ~25-30 total swipes, the preference model outperforms the engagement-only profile for recommendation quality. The blend weight α = sigmoid((n-15)/5) smoothly transitions from engagement (before swipes) to swipe-based preferences.

Core Mathematics

The key formulas powering the preference learning system.

Bradley-Terry Comparison Probability P(A preferred over B | p) = σ(pT (eA - eB)) = 1 / (1 + exp(-pT (eA - eB)))
Preference Vector Update (SGD on Log-Likelihood) pt+1 = normalize( pt + η · (1 - σ(ptT d)) · d )
where d = ewinner - eloser, η = learning rate
Bayesian Posterior Update (Laplace Approximation) Σt+1-1 = Σt-1 + λt · d · dT / σ2
where λt = σ(s)(1 - σ(s)), s = pTd
Information Gain for Pair Selection IG(i, j) = ½ log(1 + dT Σ d / σ2)
Select pair (i,j) that maximizes IG to learn most from each swipe
Warm Start: Engagement → Prior μ0 = (XTX + λI)-1 XTy     Σ0 = σnoise2 · (XTX + λI)-1
X = embedding matrix, y = normalized engagement scores
Preference Source Blending pfinal = normalize((1 - α) · pengagement + α · pswipe)
α = sigmoid((nswipes - 15) / 5)
Contrastive Preference (Like X, Not Y) p = normalize( mean(eliked) - β · mean(edisliked) )
β = 0.5 (negative signal attenuation)

Convergence Detection

The system tracks convergence at multiple levels to know when to stop asking:

  • Global: Average σ across all scenes drops below 2.5
  • Top-K: Maximum σ in top-50 scenes drops below 1.5
  • Stability: Kendall-τ rank correlation between successive re-fits exceeds 0.9
  • Pairwise: Adjacent scenes in ranking are statistically distinguishable: μA - μB > 2√(σA2 + σB2)
