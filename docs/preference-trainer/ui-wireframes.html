Three UI Approaches

Three complementary interaction patterns for preference collection, from immersive to casual.

PRIMARY

Tinder Swipe

Full-screen modal with card stack. Swipe or tap for fast binary preferences. ~8 scenes/min.

SECONDARY

Head-to-Head

Side-by-side comparison in modal. Tournament brackets. Produces true relative ranking.

CASUAL

Sidebar Quick-Rate

Fits in existing sidebar tab. Rate casually while browsing. No dedicated session needed.

Recommended: Build the Tinder Swipe as the primary experience (for dedicated training sessions of 30-50 scenes), with Sidebar Quick-Rate as a passive collection mechanism during normal browsing. The Head-to-Head mode is excellent for Phase 2 refinement within clusters.

Key Design Decisions

Tab Placement

Injected as a new "Train" tab alongside existing Analyze, Similar, and Recs tabs using the established injectSceneTabs() pattern.

Card Reuse

Uses the existing buildSceneCard() component with the new preference theme (pink/rose #ec4899).

Passive Collection

Unlike the modal approaches, the sidebar tab passively presents scenes while the user browses. No dedicated session needed. Scenes can come from:

  • The algorithm's next-best pair
  • The scene currently being viewed
  • Unwatched recommendations

Three-Button Simplicity

Nope / Skip / Like with keyboard shortcuts matching the modal. No Super-like (too complex for casual use). Precise 1-5 rating available behind a toggle for power users.

History Strip

Last 4 rated scenes shown with status icons. Tapping any item undoes that rating. Provides confidence and reversibility.

Progress Bar

Shows overall preference model confidence (from Σ convergence metrics). Motivates continued rating without being pushy.

Key Architecture Principles

1. Incremental value: Each swipe immediately improves recommendations. No "batch training" step needed.

2. Warm start: Before any swipes, the system uses engagement data. Swipes only add precision.

3. Smart pair selection: Embeddings + uncertainty guide which scenes to show. Never wastes a comparison.

4. Convergence awareness: The system knows when it has "learned enough" and tells the user.

5. Full integration: Preference data feeds directly into existing Recs tab, not a separate feature.
